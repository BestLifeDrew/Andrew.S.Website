Overview

A card game with real hardware constraints

As part of MIT MITES Summer 2025 — a six-week residential engineering program at MIT — I designed and built a fully functional Blackjack card game implemented on a microcontroller. The entire system runs on bare metal: no operating system, no high-level frameworks. All game logic, display rendering, and input handling were programmed in C, directly addressing hardware registers.

The project demanded integrating three major subsystems: a color TFT display driven over SPI, physical push-button inputs wired to GPIO pins, and a pseudo-random number generator seeded from an analog pin for card draws. Each subsystem had to work reliably together in real time.

Hardware Architecture

System components & wiring

The system was built on two linked breadboards to accommodate the wiring complexity. A microcontroller (STM32/Arduino-compatible) served as the central processing unit, with all peripherals connected via jumper wires — no PCB, no shields. This approach forced careful attention to power distribution, signal integrity, and pin conflict resolution.

Firmware & Game Logic

C programming on bare metal

The firmware was written entirely in C. The game loop handles card dealing, score calculation, dealer AI logic, and display refresh in a single-threaded execution model. Button inputs are polled each loop iteration with software debouncing — a short delay after detecting a falling edge prevents multiple triggers from mechanical bounce.

// Card draw with pseudo-random seed from analog noise int drawCard() { return (rand() % 13) + 1; // 1–13 (Ace through King) } // Dealer AI: must hit until score >= 17 while (dealerScore < 17) { dealerScore += cardValue(drawCard()); } // Debounce helper bool buttonPressed(int pin) { if (!digitalRead(pin)) { delay(50); return !digitalRead(pin); } return false; }

Display rendering was handled by writing pixel data to the TFT over SPI. Card values, score totals, and win/loss outcomes were rendered as text using a custom character drawing routine. The scoreboard visible on the display tracks multiple players' hand totals simultaneously.

Context

MIT MITES Summer 2025

MIT MITES (Minority Introduction to Engineering and Science) is a six-week residential summer program run by MIT's Office of Engineering Outreach Programs. The program admits approximately 80 students nationally each year (roughly 3% acceptance) from underrepresented backgrounds to study advanced STEM subjects on the MIT campus.

The embedded systems track covered digital logic, microcontroller architecture, C programming, and hardware interfacing. The Blackjack-21 project was the capstone build, evaluated by MIT instructors across all electronics projects in the cohort. The award for Top Electronics Project recognized the system's hardware complexity, clean firmware logic, and complete playable functionality.
