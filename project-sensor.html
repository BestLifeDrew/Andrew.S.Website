Temp & Humidity
Sensor Station

A real-time environmental monitoring station built from scratch — Arduino Uno, DHT11 sensor, 16×2 LCD, and firmware written in C++.

Overview

Building a sensor system end to end

This project was a self-directed build designed to develop hands-on experience with sensor interfacing, display protocols, and real-time data processing on a microcontroller. The goal was to build a device that continuously reads ambient temperature and humidity from a DHT11 sensor and displays live readings on a 16×2 character LCD.

Rather than using high-level shields or plug-and-play modules, everything was wired manually on a breadboard with individual jumper wires — forcing a deep understanding of how each pin, voltage rail, and data line works. The result is a clean, stable sensor station capable of displaying both temperature in Fahrenheit and relative humidity percentage simultaneously.

Hardware Architecture

Component selection & wiring

The system uses an Arduino Uno as the MCU, which is built around the ATmega328P running at 16 MHz. The DHT11 is a capacitive humidity sensor that encodes temperature and humidity data in a single-wire serial protocol — the timing of each bit is measured in microseconds, which requires careful digital pin handling in firmware.

The 16×2 LCD is driven through an I2C backpack (PCF8574 I/O expander), which reduces the connection from 8+ parallel data/control lines down to just two wires (SDA + SCL). A 10kΩ potentiometer is wired to the LCD's contrast pin, enabling physical adjustment of the display brightness.

Firmware

C++ sensor polling & display loop

The firmware uses the DHT library for sensor timing and the LiquidCrystal_I2C library for display control. The main loop reads temperature and humidity values from the sensor every 2 seconds — the DHT11's minimum sampling interval — converts temperature to Fahrenheit, and refreshes both LCD lines.

// Libraries #include <DHT.h> #include <LiquidCrystal_I2C.h> #define DHTPIN 2 #define DHTTYPE DHT11 DHT dht(DHTPIN, DHTTYPE); LiquidCrystal_I2C lcd(0x27, 16, 2); // I2C addr 0x27 void setup() { dht.begin(); lcd.init(); lcd.backlight(); } void loop() { float humidity = dht.readHumidity(); float tempC = dht.readTemperature(); float tempF = tempC * 9.0 / 5.0 + 32.0; if (!isnan(humidity) && !isnan(tempF)) { lcd.setCursor(0, 0); lcd.print("Temp: "); lcd.print(tempF, 2); lcd.print("F"); lcd.setCursor(0, 1); lcd.print("Humidity: "); lcd.print(humidity, 2); lcd.print("%"); } delay(2000); // DHT11 min sample interval }

Error handling checks for NaN (Not a Number) returns from the DHT library, which occur when the sensor fails to respond within the expected timing window. This prevents garbage data from being written to the display. The LCD auto-refreshes in place each cycle without clearing the screen first, which avoids the visible flicker that lcd.clear() would cause.

Key Learnings

What this project taught

Beyond the hardware specifics, this build developed skills that carry directly into professional embedded systems work: reading datasheets to understand sensor communication protocols, managing I2C addresses, using pull-up resistors correctly, and writing firmware that gracefully handles sensor failures.

The DHT11's single-wire protocol — where 0s and 1s are encoded as time durations rather than voltage levels — was a practical introduction to timing-sensitive digital communication, a concept that underpins protocols like UART, SPI, and I2C at the hardware level.
